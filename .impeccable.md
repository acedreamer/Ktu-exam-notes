## Design Context

### Users
KTU B.Tech students preparing for exams. They are often stressed, reading on mobile or desktop late at night, and need to quickly find "Sure Shot" topics and key theory.

### Brand Personality
**Technical Editorial**: Serious, dignified, premium. It feels like a high-end research journal or a well-maintained developer documentation site. Voice is authoritative and helpful.
**3-Word Personality**: Dignified, Utilitarian, Editorial.

### Codebase Purpose
A high-performance static Single Page Application (SPA) serving as an interactive revision dashboard for KTU engineering students (specifically CD/CST302 and IEFT/HUT300 courses). It provides dynamic notes loaded on-demand, automated LaTeX math rendering, ASCII-to-SVG graph drawer, and real-time content filtering.

### Core Goals
1. **Content is King**: Keep the reading panel central and clear, minimizing visual noise.
2. **Instant Navigation**: Fast client-side hash routing and live search filtering.
3. **Exam Efficiency**: Pre-defined priority badges to quickly identify critical material.
4. **Scroll Sync & Progress**: Real-time reading progress indicator and dynamic active topic highlights in the menu.
5. **Print Preservation**: Custom print-ready layout that hides search bars and navigation grids.

### Product's Native Shape & Core Logic
The page structure is dictated by a **nested catalog and linear, scroll-tracked reader** system:
- **Navigation Pane**: A subject/module details tree that filters on search and highlights the active reading section.
- **Reading Area**: A viewport that processes linear markdown files into styled logical card sections and intercepts mathematical expressions or ASCII drawings for rendering.
- **Scroll Sync Loop**: Continuous scroll tracking inside `#content` updating `#progress-bar` and checking topic bounding boxes.
